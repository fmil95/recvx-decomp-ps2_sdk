ARM Architecture Reference Manual

2.3 レジスタ

プロセッサには、以下に示す合計37個のレジスタがあります。

  • 30個の汎用レジスタ
  • 6個のステータス・レジスタ
  • 個のプログラム・カウンタ

レジスタは、部分的にオーバラップしているバンクに配列され、プロセッサ・モードごとにレジスタ・バンクが異なります。どの時点でも、15個の汎用レジスタ（R0〜R14）、1個または2個のステータス・レジスタ、およびプログラム・カウンタを読み出すことができます。現在読み出し可能な汎用レジスタとステータス・レジスタは、現在のプロセッサ・モードによって決まります。図2-1「レジスタの構成」に、レジスタ・バンクの構成を示します。

汎用レジスタは32ビット幅です。

レジスタ13（スタック・ポインタ；SP）は、すべてのモードでバンクされ、各モード（ユーザ・モードとR13を共有するシステム・モードを除く）に対して専用のスタック・ポインタを提供します。

レジスタ14（リンク・レジスタ；LR）は、サブルーチン・リターン・アドレス・リンク・レジスタとして使用されます。R14もすべてのモード（ユーザ・モードとR14を共有するシステム・モードを除く）でバンクされます。サブルーチン・コール（Branch and Link命令）が実行されると、R14にはサブルーチンのリターン・アドレスがセットされます。同様にR14_svc、R14_ irq、R14_fiq、R14_abort、およびR14_undefを使用して、例外発生時にリターン・アドレス（あるいは、割り込みまたは例外ルーチン内でサブルーチン・コールを実行した場合は、サブルーチンのリターン・アドレス）を保持します。R14は上記の処理以外の場合には、すべて汎用レジスタとして使用できます。

FIQモードでは（R13とR14だけでなく）レジスタR8からR12もバンクされます。高速割り込み処理（レジスタの内容をメモリにストアする必要がない割り込み処理）を可能にし、割り込みコール間で（レジスタの内容をメモリからリストアしなくてもよいように）値を保持するために、R8_fiq、R9_fiq、R10_fiq、R11_fiq、およびR12_fiqが用意されています。

レジスタR15はプログラム・カウンタ(PC)を保持します。R15を読み出すと、ビット[1:0]はゼロ、ビット[31:2]はPCの値になります。R15を書き込むと、nビット[1:0]は無視され、ビット[31:2]がPCに書き込まれます。使用方法に応じて、PCの値は命令のアドレス＋8になる場合と、予測できない場合があります。

カレント・プログラム・ステータス・レジスタ（CPSR）も、すべてのプロセッサ・モードでアクセスできます。このレジスタには、コンディション・コード・フラグ、割り込みイネーブル・フラグ、および現在のモードがあります。各特権モード（システム・モードを除く）には、例外発生時にCPSRの値を保存するのに使用するセーブされたプログラム・ステータス・レジスタ（SPSR）があります。
