ARM Architecture Reference Manual

STRT

機能説明

 ロード／ストア
アドレッシングモード アドレッシングモード2

STR{<cond>}T Rd, <post_indexed_addressing_mode>

説明

STRT (Store Register with Translation) 命令は、通常ユーザ・モードで実行するメモリ・アクセス命令をエミュレートしている（特権）例外ハンドラで使用できます。アクセスはユーザ・モード特権と同様に制限されます。

STRTは、レジスタRdから<post_indexed_addressing_mode>によって計算されるメモリのアドレスに1バイトをストアします。プロセッサが特権モードにあるときにこの命令が実行されると、そのアクセスをプロセッサがユーザ・モードにあるときと同様に処理するようメモリ・システムに通知します。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、およびUは、<addressing_mode>のタイプを指定します(セクション3.17の「アドレッシング・モード2」を参照)。

レジスタRn：<post_indexed_addressing_mode>で使用するベース・レジスタを指定します。

ユーザ・モード：この命令がユーザ・モードで実行される場合、通常のユーザ・モードのアクセスが実行されます。

R15の使用：Rdにレジスタ15を指定した場合、ストアされる値はインプリメンテーションで定義されます。

オペランド制限：RdおよびRnを同じレジスタに指定した場合、結果は予測できません。

データ・アボート：データ・アボートが通知された場合、Rnに残った値はインプリメンテーションで定義されますが、元のベース・レジスタの値か更新されたベース・レジスタの値になります。(RdとRnに同じレジスタを指定した場合でも)

アラインメント：インプリメンテーションにシステム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
