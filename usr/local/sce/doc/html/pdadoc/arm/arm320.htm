ARM Architecture Reference Manual

3.20 ロードおよびストア・マルチプル・アドレッシング・モード（代替名）

3.20.1 ブロック転送

3.18「ハーフワードのロードおよびストア、または符号付きバイトのロード・アドレッシング・モード」に記載した4つのアドレッシング・モード名（IA、IB、DA、DB）は、ロードおよびストア・マルチプル命令がブロック・データ転送に使用されているときに最も有用です。これは、ロード・マルチプルとストア・マルチプルが同じアドレッシング・モードを持つ可能性が高く、その場合にはデータはロードされたのと同じ方法でストアされるためです。

しかし、ロード・マルチプルとストア・マルチプルをスタックへのアクセスに使用している場合、ロード（ポップ）オペレーションとストア（プッシュ）オペレーションでスタックを反対方向に調整しなければならないため、データはストアに使用したのと同じアドレッシング・モードではロードされません。

3.20.2 スタック・オペレーション

ロード・マルチプルおよびストア・マルチプル・アドレッシング・モードは、スタック・オペレーションにより適した別の構文で指定することができます。スタックの記述には、以下の2種類の属性が使用されます。

フルまたはエンプティ

フル スタック・ポインタは、スタックで最後に使用された（フル）ロケーションを指します。
エンプティ スタック・ポインタは、スタックの最初の未使用（エンプティ）ロケーションを指します。

昇順または降順

昇順 下位メモリ・アドレス方向（メモリの最後尾方向に）に成長します。
降順 上位メモリ・アドレス方向（メモリの先頭方向に）に成長します。

これによって、以下の4種類のスタックを定義できます。

1. フル降順（FD）
2. エンプティ降順（ED）
3. フル昇順（FA）
4. エンプティ昇順（EA）
