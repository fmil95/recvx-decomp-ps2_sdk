ARM Architecture Reference Manual

LDRSH

機能説明

 ロード／ストア

アドレッシングモード

 アドレッシングモード3

アーキテクチャ

 アーキテクチャ v4T のみ

LDR{<cond>}SH Rd, <addressing_mode>

説明

LDRSH（Load Register Signed Hafword）命令は、適切なアドレッシング・モードと組み合わせて使用すると、16ビットの符号付きメモリ・データを汎用レジスタへロードし、汎用レジスタでその値を操作することができます。

PCをベース・レジスタとして使用すると、PC相対アドレッシングにより容易に位置独立コードを生成できます。

LDRSHは、<addressing_mode>によって計算されたメモリ・アドレスから1ハーフワードをロードし、これを32ビット・ワードに符号拡張して、レジスタRdに書き込みます。アドレスがハーフワード・アラインメントでない場合、結果は予測できません。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、U、Wビットは、<addressing_mode>のタイプを指定します（セクション3.18の「アドレッシング・モード3」を参照してください）。

addr_modeビット：これらのビットはアドレッシング・モード固有です。

レジスタRn：<addressing_mode>で使用したベース・レジスタを指定します。

R15の使用：Rdにレジスタ15が指定された場合、結果は予測できません。

オペランドの制限：<addressing_mode>がプレインデックスまたはポストインデックス付きアドレッシングを使用し、RdとRnに同じレジスタが指定されている場合、結果は予測できません。

データ・アボート：データ・アボートが通知され、<addressing_mode>がプレインデックスまたはポストインデックス付きアドレッシングを使用している場合、Rnに残った値は、インプリメンテーションで定義されますが（RdとRnに同じレジスタが指定されている場合でも）元のベース・レジスタ値か更新されたベース・レジスタ値になります。

ハーフワードに整列していないアドレス：ロード・アドレスがハーフワードに整列していない場合、ロードされた結果は予測できません。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[0]!=0のアドレスによって、アラインメント例外が発生します。
