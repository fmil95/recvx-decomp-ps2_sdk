ARM Architecture Reference Manual

ADD

機能説明

 データ処理

アドレッシングモード

 アドレッシング・モード１

ADD{<cond>}{S} Rd, Rn, <shifter_operand>

説明

ADD命令は、<shifter_operand>の値をレジスタRnの値に加算し、結果をデスティネーション・レジスタRdにストアします。（結果に基づいて）コンディション・コード・フラグが更新されます。

ADDは2つの値を加算して、第3の値を生成するのに使用します。

（Rxの）レジスタ値をインクリメントするには、以下の式を使用します。

ADD Rx, Rx, #1

（Rxの値に）2n+1を定数乗算して（結果をRdに格納）は、以下で実行できます。

ADD Rx, Rx LSL #n

PC相対アドレスを形成するには、以下の式を使用します。

ADD Rs, PC, #offset

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = Rn + <shifter_operand>
        if S == 1 and Rd == R15 then

例外

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、Z、C、Vを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16以降の「アドレッシング・モード1」で説明しています。

Iビット：ビット25を使用して、<shift_operand>のイミディエイト形式とレジスタ形式を区別することができます。

R15への書き込み：RdがR15で、命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15で、Sフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・コードでは予測不能です。
