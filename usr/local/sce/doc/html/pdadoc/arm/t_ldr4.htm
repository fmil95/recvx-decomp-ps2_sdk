ARM Architecture Reference Manual

LDR（4）

機能説明

 Load word SP-relative

アーキテクチャ

 アーキテクチャ v4T のみ

LDR Rd, [SP, #<8_bit_offset> * 4]

説明

この形式のLDR（Load Register）命令は、32ビットのメモリ・データを汎用レジスタにロードし、その値を操作することができます。このアドレッシング・モードは、スタック・データにアクセスするのに便利です。

この場合、LDRはメモリから1ワードをロードし、それをレジスタRdに書き込みます。メモリ・アドレスは、<8_bit_offset>の値×4をSPの値に加算して計算されます。アドレスがワードに整列していない場合、結果は予測できません。

オペレーション

<address> = SP + (8_bit_offset * 4)
if <address>[1:0] == 0b00
	<data> = Memory[<address>,4]
else
	<data> = UNPREDICTABLE
Rd = <data>

データ・アボート

修飾子

なし

注記

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章「システム・アーキテクチャとシステム・コントロール・コプロセッサ」を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
