ARM Architecture Reference Manual

STR

機能説明

 ロード／ストア・マルチプル
アドレッシングモード アドレッシングモード2

STR{<cond>} Rd, <addressing_mode>

説明

STR (Store register) 命令は、適切なアドレッシング・モードと組み合わせることにより、汎用レジスタの32ビット・データをメモリにストアします。PCをベース・レジスタとして使用すれば、PC相対アドレッシングが可能であり、容易に位置独立コードを生成できます。

STRは、レジスタRdの1ワードを<addressing_mode>で計算されたメモリ・アドレスにストアします。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、U、およびWビットで<addressing_mode>のタイプを指定します（セクション3.17の「アドレッシング・モード2」を参照）。

レジスタRn：<addressing_mode>で使用するベース・レジスタを指定します。

R15の使用：Rdにレジスタ15を指定した場合、ストアされる値はインプリメンテーションで定義されます。

オペランドの制限：<addressing_mode>でプレインデックスまたはポストインデックス・アドレッシングを使用して、RdとRnに同じレジスタを指定した場合、結果は予測できません。

データ・アボート：データ・アボートが通知され、<addressing_mode>でプレインデックスまたはポストインデックス・アドレッシングを使用している場合、Rnに残る値はインプリメンテーションで定義されますが、元のベース・レジスタの値または更新されたベース・レジスタの値のいずれかになります（RdとRnに同じレジスタを指定した場合でも）。

アラインメント：インプリメンテーションにシステム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
