ARM Architecture Reference Manual

ADC

機能説明

 データ処理

アドレッシングモード

 アドレッシング・モード１

ADC{<cond>}{S} Rd, RN,<shifter_operands>

説明

ADC（Add with Carry）命令は、<shifter_operand>の値とキャリ・フラグの値をレジスタRnの値に加算し、結果をデスティネーション・レジスタRdにストアします。オプションにより、（結果に基づいて）コンディション・コード・フラグが更新されます。

ADCは複数ワードの加算を合成するのに使用します。レジスタ・ペアR0、R1およびR2、R3が64ビット値を保持している（R0とR2は最下位ワードを保持）場合、以下の命令によってR4、R5に64ビットの和が格納されます。

ADDS	R4,R0,R2
ADC	R5,R1,R3

また、以下の命令、

ADCS R0,R0,R0

は、R0に対して1ビットのRotate Left with Extendオペレーション（キャリ・フラグを含む33ビット・ローテート）を実行します。詳細については、アドレッシングモード１"Rm, RRXを参照してください。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、Z、C、Vを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16以降の「アドレッシング・モード1」で説明しています。

Iビット：ビット25を使用して、<shifter_operand>のイミディエイト形式とレジスタ形式を区別することができます。

R15への書き込み：RdがR15で、命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15で、Sフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。
