ARM Architecture Reference Manual

LDRT

機能説明

 ロード／ストア

アドレッシングモード

 アドレッシングモード2

LDR{<cond>}T Rd, <post_indexed_addressing_mode>

説明

LDRT（Load Register with Translation）命令は、通常ユーザ・モードで実行するメモリ・アクセス命令をエミュレートする（特権）例外ハンドラで使用することができます。このアクセスは、ユーザ・モード特権を持っている場合と同じように制限されます。

LDRTはメモリ・アドレスから1ワードをロードし、これをレジスタRdに書き込みます。プロセッサが特権モードの場合にこの命令を実行すると、メモリ・システムに対し、プロセッサがユーザ・モードのときと同じようにこのアクセスを処理するよう通知されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        if <address>[1:0] == 0b00
                Rd = Memory[<address>,4]
        else if <address>[1:0] == 0b01
                Rd = Memory[<address>,4] Rotate_Right 8
        else if <address>[1:0] == 0b10
                Rd = Memory[<address>,4] Rotate_Right 16
        else /* <address>[1:0] == 0b11 */
                Rd = Memory[<address>,4] Rotate_Right 24

注記

アドレッシング・モード：I、P、Uビットは、<addressing_mode>のタイプを指定します（セクション3.17の「アドレッシング・モード2」を参照してください）。

レジスタRn：<post_indexed_addressing_mode>で使用したベース・レジスタを指定します。

ユーザ・モード：この命令がユーザ・コードで実行されると、通常のユーザ・コードのアクセスが行われます。

オペランドの制限：RdとRnに同じレジスタが指定された場合、結果は予測できません。

データ・アボート：データ・アボートが通知された場合、Rnに残った値はインプリメンテーションで定義されますが、（RdとRnに同じレジスタが指定されている場合でも）元のベース・レジスタ値か更新されたベース・レジスタ値になります。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
