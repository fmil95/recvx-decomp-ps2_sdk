ARM Architecture Reference Manual

LDM（1）

機能説明

 ロード／ストアマルチプル

アドレッシングモード

 アドレッシングモード4

LDM{<cond>}<addressing_mode> Rn{!}, <registers>

説明

この形式のLDM（Load Multiple）命令は、ブロック・ロード命令として有用で（ストア・マルチプル命令と組み合わせると、効果的にブロックをコピーできます）、プロシージャの終了時を含むスタック操作で、セーブしたレジスタのリストア、リターン・アドレスのPCへのロード、スタック・ポインタの更新を行うのに便利です。

この場合、LDMはシーケンシャル・メモリ・ロケーションから汎用レジスタの空でないサブセット（またはすべて）をロードします。レジスタは、最小番号のレジスタを先頭にして最下位メモリ・アドレス（<start_addr>）から、そして最大番号のレジスタを最後にして最上位メモリ・アドレス（<end_addr>）から、という順序でロードされます。PCがレジスタ・リストで指定されている（オペコードのビット15がセットされている）場合、この命令によってPCにロードされたアドレス（データ）への分岐が発生します。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        <address> = <start_address>
        for i = 0 to 15
                if <register_list>[i] == 1
                        Ri = Memory[<address>,4]
                        <address> = <address> + 4
       assert <end_address> == <address> - 4

例外

データ・アボート

修飾子

コンディション・コード
!はWビットをセットし、ベース・レジスタを更新します。

注記

アドレッシング・モード：P、U、Wビットは、異なるタイプのアドレッシング・モードを区別します。セクション3.19「アドレッシング・モード4」を参照してください。

レジスタRn：<addressing_mode>で使用したベース・レジスタを指定します。

R15の使用：R15をベース・レジスタRnとして使用した場合、結果は予測できません。

オペランドの制限：ベース・レジスタRnが<register_list>に指定され、かつライトバックが指定されている場合、Rnの最終値は予測できません。

データ・アボート：データ・アボートが通知され、<addressing_mode>がライトバックを指定する場合、Rnに残った値はインプリメンテーションで定義されますが、（<register_list>でRnが指定されていた場合でも）元のベース・レジスタ値か更新されたベース・レジスタ値になります。レジスタ15が<register_list>に指定されている場合、データ・アボートが発生した場合にこれに上書きしてはなりません。

ワードに整列していないアドレス：ロード・マルチプル命令は、<address>の最下位2ビットを無視します（ワードはLoad Wordのようにローテートされません）。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
