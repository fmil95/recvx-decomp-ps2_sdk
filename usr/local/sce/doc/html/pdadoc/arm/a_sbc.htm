ARM Architecture Reference Manual

SBC

機能説明

 データ処理
アドレッシングモード アドレッシングモード１

SBC{<cond>}{S} Rd, Rn, <shifter_operand>

説明

SBC (Subtract with Carry)命令は、複数ワードの減算を合成するために使用されます。R0とR1およびR2とR3のレジスタのペアが64ビット値を保持している場合（R0とR2が最下位ワードを保持）、以下の命令は64ビットの差をR4、R5に置きます。

SUB  R4, R0, R2
SBC  R5, R1, R3

SBCはレジスタRnの値から<shifter_operand>とNOT (Carry Flag) の値を減算し、結果をデスティネーション・レジスタRdにストアします。コンディション・コード・フラグは、オプションにより（結果に基づいて）更新されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = Rn - <shifter_operand> - NOT(C Flag)
        if S == 1 and Rd == R15 then
                CPSR = SPSR
        else if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = NOT BorrowFrom(Rn - <shifter_operand> - NOT(C Flag))
                V Flag = OverflowFrom(Rn - <shifter_operand> - NOT(C Flag))

例外

なし

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、Z、CおよびVを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16の「アドレッシング・モード1」で説明しています。

R15への書き込み：RdがR15で命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15でSフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

Iビット：ビット25を使用して、<shifter_operand> のイミディエイト形式とレジスタ形式を区別することができます。
