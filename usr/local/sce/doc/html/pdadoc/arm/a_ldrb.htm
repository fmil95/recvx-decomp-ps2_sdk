ARM Architecture Reference Manual

LDRB

機能説明

 ロード／ストア

アドレッシングモード

 アドレッシングモード2

LDR{<cond>}B Rd, <addressing_mode>

説明

LDRB（Load Register Byte）命令は、適切なアドレッシング・モードと組み合わせると、8ビットのメモリ・データを汎用レジスタにロードし、汎用レジスタでその値を操作することができます。PCをベース・レジスタとして使用すると、PC相対アドレッシングにより、容易に位置独立コードを生成することができます。

LDRBは<addressing_mode>によって計算されたメモリ・アドレスから1バイトをロードし、このバイトを32ビット・ワードにゼロ拡張してレジスタRdに書き込みます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、U、Wビットは、<addressing_mode>のタイプを指定します（セクション3.17の「アドレッシング・モード2」を参照してください）。

レジスタRd：<addressing_mode>で使用されたベース・レジスタを指定します。

R15の使用：Rdにレジスタ15が指定されている場合、結果は予測できません。

オペランドの制限：<addressing_mode>がプレインデックスまたはポストインデックス付きアドレッシングを使用し、RdとRnに同じレジスタが指定されている場合、結果は予測できません。

データ・アボート：データ・アボートが通知され、<addressing_mode>がプレインデックスまたはポストインデックス付きアドレッシングを使用している場合、Rnに残った値はインプリメンテーションで定義されていますが、（RdとRnに同じレジスタが指定されている場合でも）元のベース・レジスタ値か更新されたベース・レジスタ値になります。
