SUB

機能説明

 データ処理
アドレッシングモード アドレッシングモード１

SUB{<cond>}{S} Rd, Rn, <shifter_operand>

説明

SUB (Subtract) 命令は、ある値を別の値から減算して第3の値を生成するために使用されます。(Rxにある)レジスタの値をデクリメントするには、以下の構文を使用します。

SUB Rx、Rx、#1

SUBは、レジスタRnの値から<shifter_operand>の値を減算し、結果をデスティネーション・レジスタRnにストアします。コンディション・コード・フラグはオプションにより、(結果に基づいて)更新されます。

SUBSはCMP Rx, #0がなくても、ループ分岐がフラグを見て適切な終了条件をテストできるため、ループ・カウンタ・デクリメントとして便利です。

割り込みからリターンするには、SUBS PC, LR, #4を使用してください。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = Rn - <shifter_operand>
        if S == 1 and Rd == R15 then
                CPSR = SPSR
        else if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = NOT BorrowFrom(Rn - <shifter_operand>)
                V Flag = OverflowFrom(Rn - <shifter_operand>)

例外

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、ZおよびＶを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16の「アドレッシング・モード1」を参照してください。

R15への書き込み：RdがR15で命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15でSフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

Iビット：ビット25を使用して、<shifter_operand> のイミディエイト形式とレジスタ形式を区別することができます。
