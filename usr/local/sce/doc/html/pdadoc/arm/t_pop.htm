ARM Architecture Reference Manual

POP

機能説明

 Pop multiple registers

アーキテクチャ

 アーキテクチャ v4T のみ

POP {<register_list>, <PC>}

説明

POP（Pop Multiple Registers）命令は、プロシージャ終了を含むスタック操作で、保存したレジスタのリストア、PCへのリターン・アドレスのロード、スタック・ポインタの更新を行うのに便利です。

POPは、汎用レジスタのサブセット（または完全なセット）をロードし、オプションで順次メモリ・ロケーションからPCをロードします。レジスタは以下の順序でロードされます。

  • 最小番号のレジスタを先頭に、最下位メモリ・アドレス（<start_address>）から
  • 最大番号のレジスタを最後に、最上位メモリ・アドレス（<end_address>）から

<start_address>はSPの値です。

これ以降のアドレスは、前のアドレスを4ずつインクリメントして形成されます。<register_list>で指定された各レジスタに対して、1つのアドレスが生成されます。

<end_address>の値は、SPの値と<register_list>のレジスタ数（Rビットを含む）×4の合計から4を引いたものになります。
最後にベース・レジスタRnが、<register_list>のレジスタ数（＋Rビット）×4だけインクリメントされます。

オペレーション

<start_address> = SP
<end_address> = SP + 4*(R + Number_Of_Set_Bits_In(<register_list>))
<address>=<start_address>
for i = 0 to 7
	if <register_list>[i] == 1
		Ri = Memory[<address>,4]
		<address> = <address> + 4
if R == 1
	PC = Memory[<address>,4]
	<address> = <address> + 4
assert <end_address> == <address> - 4
SP = SP + 4*(R + Number_Of_Set_Bits_In(<register_list>))

例外

データ・アボート

修飾子

なし

注記

Rビット：R==1の場合、PCもスタックからロードされます。R==0の場合、PCはロードされません。

データ・アボート：データ・アボートが通知された場合、SPに残った値はインプリメンテーションで定義されますが、元のSP値か変更されたSP値になります。

ワードに整列していないアドレス：Pop multiple命令は、<address>の最下位2ビットを無視します（ワードはLoad Wordのようにローテートされません）。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章「システム・アーキテクチャとシステム・コントロール・コプロセッサ」を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
