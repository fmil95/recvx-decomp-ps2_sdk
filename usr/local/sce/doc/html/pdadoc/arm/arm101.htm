ARM Architecture Reference Manual

1.1 概要

ARMは、以下に示す一般的なRISC（Reduced Instruction Set Computer：縮小命令セット・コンピュータ）アーキテクチャの機能をすべて備えています。

  • 大きな均一レジスタ・ファイル
  • ロード/ストア・アーキテクチャ（レジスタの内容だけを操作するデータ処理オペレーション）
  • シンプルなアドレッシング・モード（レジスタおよび命令フィールドで指定されるアドレスからデータをロード/ストア）
  • 均一および固定長命令フィールド（命令のデコードを単純化）

さらに、ARMアーキテクチャには以下のような機能があります。

  • データ処理命令ごとにALUとシフタの両方を制御することにより、シフタとALUを最大限に活用
  • オート・インクリメントおよびオート・デクリメント・アドレッシング・モードにより、プログラムのループを最適化
  • ロードおよびストア・マルチプル命令により、データ・スループットを最大化
  • 全命令の条件付き実行により、実行スループットを最大化

基本的なRISCアーキテクチャに加えて、これらの機能が強化されたことにより、高性能、低消費電力、最小ダイ・サイズのバランスの取れたインプリメンテーションを実現できます。

1.1.1 ARMレジスタ

ARMには、31個の32ビット・レジスタがあります。どの時点でも16個のレジスタが読み出し可能であり、残りのレジスタは例外処理の高速化に使用されます。ARM命令のすべてのレジスタ指定子で、16個のレジスタのいずれかをアドレス指定することができます。

16個のレジスタのメイン・バンクは、すべて非特権コードによって使用されます。すなわち、これらはユーザ・モード・レジスタです。他のすべてのモードとは異なり、ユーザ・モードは非特権モードです。このことは、ユーザ・モードが、（例外を発生しないで）別のプロセッサ・モードに切り替えることができない唯一のモードであることを意味します。

プログラム・カウンタ

レジスタ15はプログラム・カウンタ（PC）です。大部分の命令で、実行中の命令の2つ後の命令を指すポインタとして使用できます。ARM命令はすべて4バイト長（1つの32ビット・ワード）で、常にワード境界に整列するため、PCにはちょうど30ビットが含まれ、最下位2ビットは常にゼロになります。

リンク・レジスタ

レジスタ14はリンク・レジスタ（LR）と呼ばれます。リンク・レジスタの機能は、Branch with Link（BL）命令に続く命令のアドレスを保持することです。BL命令はサブルーチン・コールに使用されます。それ以外の場合、R14は汎用レジスタとして使用できます。

その他のレジスタ

残り14個のレジスタには、ハードウェア上の特別な目的はなく、それらの用途は純粋にソフトウェアによって定義されます。ソフトウェアは通常R13をスタック・ポインタ（SP）として使用します。
