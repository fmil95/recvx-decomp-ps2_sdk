3.14 命令セットの拡張

ARM命令セットは次の4つの領域で拡張できます。

  • 算術命令の拡張スペース
  • 制御命令の拡張スペース
  • ロード/ストア命令の拡張スペース
  • コプロセッサ命令の拡張スペース

現在、これらの命令は未定義です（これらは未定義の命令例外を発生します）。これらアドレス空間は、今後新しい命令の追加に使用されます。

3.14.1 算術命令の拡張スペース

次のオペコードでの命令は算術命令の拡張スペースです。

opcode[27:24]	=	0000
opcode[7:4]	=	0b1001

指定されたフィールド名はガイドラインにすぎません。これは実装を容易にすると思われます。

MULとMLA

乗算と積和演算（MULとMLA）命令はop1値0〜3を使用します。
Rn デスティネーション・レジスタを指定します。
RmとRs ソース・レジスタを指定します。
Rd 累積値を指定します。

UMULL、UMLAL、SMULL、SMLAL

符号付き/符号なし複数ロングと乗算累積ロング（UMULL、UMLAL、SMULL、SMLSL）命令はop1値8〜15を使用します。
RnとRd デスティネーション・レジスタを指定します。
RmとRs ソース・レジスタを指定します。

その他のオペコード

算術命令の拡張スペースにおけるその他全オペコードの意味は次のとおりです。

  • ARMアーキテクチャ4では未定義
  • アーキテクチャ初期バージョンでは予測不可能

3.14.2 制御命令の拡張スペース

opcode[27:26]	=	0b00

opcode[20]  	=	0

この条件を満たし、かつ、

opcode[25]	=	0
opcode[7]	=	1
opcode[4]	=	1

でない命令は、制御命令の拡張スペースです。指定されたフィールド名は、実装を容易にすると思われるガイドラインにすぎません。

MRS

汎用レジスタ（MRS）へのムーブ・ステータス・レジスタ命令は、以下の通りです。

opcode	 [25]	=	0
opcode	[19:16]	=	0b1111
opcode[11:0]	     =	0

op1=0b00とop1=0b10の両方を使用します。
Rdはデスティネーション・レジスタを指定します。

MSR

汎用レジスタをステータス・レジスタへ移動する命令（MSR）は、以下の通りです。

opcode[25] 	=	0
opcode[15:12]	=	0b1111
opcode[11:4]	=	0

op1=0b01とop1=0b11の両方を使用します。
Rm ソース・レジスタを指定します。
opcode[19:16] ライト・マスクを保持します。

イミディエイト値をステータス・レジスタ（MSR）にムーブする命令は、以下の通りです。

opcode[25] 	=	1
opcode[15:12]	=	0b1111

また、op1 = 0b01とop1 ＝ 0b11の両方を使用します。
Rm ソース・レジスタを指定します。
opcode[11:0] イミディエイト・オペランドを指定します。
opcode[19:16] ライト・マスクを保持します。

BX

分岐と交換命令セット（BX）命令は、次をセットは以下の通りです。

opcode[25]	     =	0
opcode[19:8]	=	0b111111111111
opcode[7:4]	=	0b0001

また、 op1 = 0b01を使用します。
Rmを使用してソース・レジスタを指定します。

その他のオペコード

制御命令の拡張スペースにおけるその他全オペコードの意味は、次のとおりです。
ARMアーキテクチャ4では未定義
アーキテクチャの初期バージョンでは予測不可能

3.14.3 ロード/ストア命令の拡張スペース

opcode[27:25]  =	0b000

opcode[24] 	=	0

SWPとSWPB

スワップとスワップ・バイト（SWPとSWPB）命令セットは以下の通りです。

opcode[24:23]	=	0b10
opcode[21:20]	=	0b00
opcode[11:4]	=	0b00001001

構文説明：

Rn ベース・アドレスを指定します。
Rd デスティネーション・レジスタを指定します。
Rm ソース・レジスタを指定します。
Rd デスティネーションを指定します。
Opcode[22] バイト転送を示します。

LDRH

ハーフワードのロード（LDRH）命令は以下の通りです。

opcode[20]	=	1
op1	     =	0b01

構文説明：

Bビット レジスタとイミディエイト・オフセットを区別します。
P、U、Wビット アドレッシング・モードを指定します。
Rn ベース・レジスタを指定します。
Rm レジスタ・オフセットを指定します。
RsとRm イミディエイト・オフセットを指定します。

LDRSH

符号付きハーフワードのロード（LDRSH）命令は以下の通りです。

opcode[20]	=	1
op1		=	0b11

構文説明：

Bビット レジスタとイミディエイト・オフセットを区別します。
P、U、Wビット アドレッシング・モードを指定します。
Rn ベース・レジスタを指定します。
Rd デスティネーション・レジスタを指定します。
Rm レジスタ・オフセットを指定します。
RsとRm イミディエイト・オフセットを指定します。

LDRSB

符号付きハーフワードのロード（LDRSB）命令は以下の通りです。

opcode[20]	=	1
op1		=	0b10

構文説明：

Bビット レジスタとイミディエイト・オフセットを区別します。
P、U、Wビット アドレッシング・モードを指定します。
Rn ベース・レジスタを指定します。
Rd デスティネーション・レジスタを指定します。
Rm レジスタ・オフセットを指定します。
RsとRm イミディエイト・オフセットを指定します。

STRH

ハーフワードのストア（STRH）命令は以下の通りです。

opcode[20] = 0
op1 = 0b01

構文説明：

Bビット レジスタとイミディエイト・オフセットを区別します。
P、U、Wビット アドレッシング・モードを指定します。
Rn ベース・レジスタを指定します。
Rd ソース・レジスタを指定します。
Rm レジスタ・オフセットを指定します。
RsとRm イミディエイト・オフセットを指定します。

その他のオペコード

ロード/ストア命令の拡張スペースにおけるその他全オペコードの意味は次のとおりです。

  • ARMアーキテクチャ4では未定義
  • アーキテクチャの初期バージョンでは予測不可能
