ARM Architecture Reference Manual

STM（2）

機能説明

 ロード／ストア・マルチプル
アドレッシングモード アドレッシングモード４

STM{<cond>}<addressing_mode> Rn{!}, <registers>^

説明

STM (Store Multiple) 命令は、プロセッサが特権モードにあるときに、ユーザ・モード・レジスタをストアするのに使用されます（プロセス・スワップを実行するときに便利です）。

この形式のSTMは、ユーザ・モードの汎用レジスタ（システム・モード汎用レジスタでもある）のサブセット（または全セット）を順次メモリ・ロケーションにストアします。レジスタは、最小番号レジスタを先頭に最下位メモリ・アドレス（<start_address>）から順次ストアされ、最後に最大番号レジスタが最上位アドレス(<end_address>)にストアされます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレシング・モード：PおよびWビットは、異なるタイプのアドレッシング・モードを区別します。セクション3.19の「アドレッシング・モード4」を参照してください。

バンクされたレジスタ：この命令の後、バンクされたレジスタにアクセスする命令が続いてはなりません（これを確実にするために、後にNOPを置くようにしてください）。

ライトバック：ビット21（Wビット）セットした場合、結果は予測できません。

ユーザ・モードとシステム・モード：この命令は、ユーザ・モードおよびシステム・モードでは予測できません。

レジスタRn：<addressing_mode>で使用するベース・レジスタを指定します。

R15の使用：レジスタ15がベース・レジスタRnとして指定された場合、結果は予測できません。レジスタ15が<registe_list>で指定された場合、ストアされる値はインプリメンテーションで定義されます。

ベース・レジスタ・モード：ベース・レジスタは、ユーザ・モード・レジスタではなく、現在のプロセッサ・モード・レジスタから読み込まれます。

データ・アボート：データ・アボートが通知される場合、Rnに残る値はもとのベース・レジスタになります。

ワードに整列していないアドレス：ストア・マルチプル命令は、<address>の最下位2ビットを無視します（ワードはLoad Word命令のようにローテートされません）。

アライメント：インプリメンテーションにシステム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
