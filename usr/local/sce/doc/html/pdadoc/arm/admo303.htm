ARM Architecture Reference Manual

Addressing Mode 3

機能説明

 Immediate pre-indexed
アーキテクチャ アーキテクチャv4 のみ

[Rn, #+/-<8_bit_offset>]!

説明

このアドレッシング・モードにより、ポインタ値の自動更新を伴う配列へのポインタ・アクセスを行うことができます。

ベース・レジスタRnの値にイミディエイト・オフセットの値を加算するか、またはベース・レジスタRnからイミディエイト・オフセットの値を減算してアドレスを計算します。

命令で指定された条件がコンディション・コードのステータスと一致した場合、計算されたアドレスはベース・レジスタRnに書き戻されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

<8_bit_offset> = (<immedH> << 4) OR <immedL>
if U == 1 then
        <address> = Rn + <8_bit_offset>
else /* U == 0 */
        <address> = Rn - <8_bit_offset>
if ConditionPassed(<cond>) then
        Rn = <address>

注記

Lビット：ロード命令(L==1)とストア命令(L==0)を区別します。

Sビット：符号付きハーフワード・アクセス(S==1)と符号なしハーフワード・アクセス(S==0)を区別します。

Hビット：ハーフワード・アクセス(H==1)と符号付きバイト・アクセス(H==0)を区別します。

R15の使用：レジスタRnとしてR15を指定した場合、結果は予測できません。
