ARM Architecture Reference Manual

3.17 ワードまたは符号なしバイト・ロード/ストア・アドレッシングモード

ワードまたは符号なしバイト・ロード/ストア命令のアドレス計算に使用するアドレッシング・モードが9つあります。各アドレッシング・モードの詳細について、以降説明します。

イミディエイト・オフセット

LDR|STR{<cond>}{B}     Rd, [Rn, #+/-<12_bit_offset>]

レジスタ・オフセット

LDR|STR{<cond>}{B}     Rd, [Rn, +/-Rm]

スケール付きレジスタ・オフセット

LDR|STR{<cond>}{B}     Rd, [Rn, +/-Rm, <shift> #<shift_imm>]

イミディエイト・プレインデックス

LDR|STR{<cond>}{B}     Rd, [Rn, #+/-<12_bit_offset>]!

レジスタ・プレインデックス

LDR|STR{<cond>}{B}     Rd, [Rn, +/-Rm]!

スケール付きレジスタ・プレインデックス

LDR|STR{<cond>}{B} Rd, [Rn, +/-Rm, <shift> #<shift_imm>]!

イミディエイト・ポストインデックス

LDR|STR{<cond>}{B}{T}   Rd, [Rn], #+/-<12_bit_offset>

レジスタ・ポストインデックス

LDR|STR{<cond>}{B}{T}    Rd, [Rn], +/-Rm

スケール付きレジスタ・ポストインデックス

LDR|STR{<cond>}{B}{T}    Rd, [Rn], +/-Rm, <shift> #<shift_imm>

注記

Pビット：プレ/ポスト・インデックス操作：

プレインデックス操作(P==1)は、ベース・レジスタにオフセットを適用し、結果をアドレスとして使用することを示します。

ポストインデックス操作(P==0)は、ベース・レジスタ値をアドレスに使用し、次にベース・レジスタにオフセットを適用して、ベース・レジスタに書き戻すことを示します。

Uビット：オフセットをベースに加算する（U==1）か、またはベースから減算する（U==0）ことを示します。

Bビット：符号なしバイト・アクセス（B==1）とワード・アクセス（B==0）を区別します。

Wビット：このビットには以下の2つの意味があります。

P==1の場合 W==1の場合、計算されたアドレスがベース・レジスタに書き戻されます（W==0の場合、ベース・レジスタは更新されません。）
P==0の場合 W==1の場合、現在のアクセスは、（保護およびメモリ・システムによって）ユーザ・モード・アクセスとして扱われます(W==0の場合、通常のアクセスが実行されます）。

Lビット：ロード(L==1)とストア(L==0)を区別します。
