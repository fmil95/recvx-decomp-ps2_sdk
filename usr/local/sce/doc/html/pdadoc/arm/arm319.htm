ARM Architecture Reference Manual

3.19 ロードおよびストア・マルチプル・アドレッシング・モード

ロード・マルチプル命令は、メモリから汎用レジスタのサブセット（または全セット）をロードします。ストア・マルチプル命令は、汎用レジスタのサブセット（または全セット）をメモリにストアします。これらの命令の命令形式は1つです。

ロードおよびストア・マルチプル・アドレッシング・モードは、連続した範囲のアドレスを生成します。最小番号のレジスタは最下位メモリ・アドレスに、また最大番号のレジスタは、最上位メモリ・アドレスにそれぞれストアされます。

以下の4つのロードおよびストア・マルチプル・アドレッシング・モードがあります：

1. インクリメント・アフタ

LDM|STM{<cond>}IA   Rn{!}, <registers>{^}

2. インクリメント・ビフォ

LDM|STM{<cond>}IＢ   Rn{!}, <registers>{^}

3. デクリメント・アフタ

LDM|STM{<cond>}DA   Rn{!}, <registers>{^}

4. デクリメント・ビフォ

LDM|STM{<cond>}DB   Rn{!}, <registers>{^}

注記

レジスタ・リスト：<register_list>には、各汎用レジスタに対して1ビットがあり、ビット0はレジスタ0、ビット15はレジスタ15（PC）に対応しています。
<register_list>は、レジスタをコンマで区切り括弧でくくったレジスタ・リストを使用して、命令ニーモニックに指定されています。どのビットもセットされていない場合、結果は予期できません。

Uビット：ベース・レジスタから上方（U==1）または下方（U==0）に転送が行われることを示します。

Pビット：プレインデックスまたはポストインデックス操作：

P== 1 メモリ・アクセスに使用する前に、範囲内の各アドレスがインクリメント(U==1)またはデクリメント(U==0)されることを示します。
P== 0 メモリ・アクセスに使用した後に、範囲内の各アドレスがインクリメント(U==1)またはデクリメント(U==0)されることを示します。

Wビット：転送後にベース・レジスタが更新されることを示します。ベース・レジスタは、レジスタ・リストのレジスタ数×4だけインクリメント（U==1）またはデクリメント（U==0）されます。

Sビット：PCをロードするLDMの場合、SビットはCPSRがSPSRからロードされることを示します。PCをロードしないLDM、および、すべてのSTMの場合は、Sビットはプロセッサが特権モードにあり、現在のモードのレジスタではなく、ユーザ・モードでバンクされたレジスタが転送されることを示します。

Lビット：ロード(L==1)命令とストア(L==0)命令を区別します。
