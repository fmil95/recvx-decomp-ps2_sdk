ARM Architecture Reference Manual

STRH

機能説明

 ロード／ストア
アドレッシングモード アドレッシングモード3
アーキテクチャ アーキテクチャｖ４のみ

STR{<cond>}H Rd, <addressing_mode>

説明

STRH (Store Register Halfword) 命令は、適切なアドレッシング・モードと組み合わせることにより、16ビットのデータを汎用レジスタからメモリにストアすることができます。PCをベース・レジスタとして使用すれば、PC相対アドレッシングが可能であり、容易に位置独立コードを生成できます。

STRHは、レジスタRdの最下位ハーフワードから<addressing_mode>によって計算されるメモリのアドレスにハーフワードをストアします。アドレスがハーフ・ワードに整列していない場合、結果は予測不能になります。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        if <address>[0] == 0
                <data> = Rd[15:0]

                <data> = UNPREDICTABLE

例外

データ・アボートI、P、およびWビットは<addressing_mode>のタイプを指定します。(「アドレッシング・モード3」を参照。)

注記

アドレッシング・モード：I、P、U、およびWビットは、<addressing_mode>のタイプを指定します(「アドレッシング・モード3」を参照)。

addr_modeビット：これらのビットはアドレッシング・モード固有です。

レジスタRn：<addressing_mode>を使用するベース・レジスタを指定します。

R15の使用：Rdにレジスタ15を指定した場合、結果は予測できません。

オペランドの制限：<addressing_mode>がプレインデックスまたはポストインデックス・アドレッシングを使用している場合、および同じレジスタをRdおよびRnに指定した場合、結果は予測できません。

データ・アボート：データ・アボートが通知された場合、および<addressing_mode>がプレインデックスおよびポストインデックス・アドレッシングを使用している場合、Rnに残った値はインプリメンテーションで定義されますが、元のベース・レジスタの値か更新されたベース・レジスタの値になります。(RdとRnに同じレジスタを指定した場合でも)

ハーフワードに整列していないアドレス：ストア・アドレスがハーフワード整列していない場合、ストアされる値は予測できません。

アラインメント：インプリメンテーションにシステム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
