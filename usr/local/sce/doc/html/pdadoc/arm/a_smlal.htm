ARM Architecture Reference Manual

SMLAL

機能説明

 乗算
アーキテクチャ アーキテクチャｖ３、ｖ４のみ

SMLAL{<cond>}{<S>} RdLo, RdHi, Rm, Rs

説明

SMLAL (Signed Multiply Accumulate Long) 命令は、符号付き変数を乗算して64ビットの結果を生成し、2つのデスティネーション汎用レジスタの64ビット値に加算します。結果は2つのデスティネーション汎用レジスタに書き戻されます。

SMLALは、レジスタRmの符号付きの値とレジスタRsの符号付きの値を乗算し、64ビットの結果を生成します。結果の下位32ビットはRdLoに加算され、RdLoにストアされます。上位32ビットとRdLoへの加算からのキャリがRdHiに加算され、RdHiにストアされます。コンディション・コード・フラグは、オプションにより（64ビットの結果に基づいて）更新されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        RdLo = (Rm * Rs)[31:0] + RdLo /* Signed multiplication */
        RdHi = (Rm * Rs)[63:32] + RdHi + CarryFrom((Rm * Rs)[31:0] + RdLo)
        if S == 1 then
                N Flag = RdHi[31]
                Z Flag = if (RdHi == 0) and (RdLo == 0) then 1 else 0
                C Flag = unpredictable
                V Flag = unpredictable

例外

なし

修飾子

コンディション・コード
Sはコンディション・コード・フラグNとZを更新します。

注記

R15の使用：レジスタRdHi、RdLo、Rm、またはRsにR15を指定した場合、結果は予測できません。

オペランドの制限：RdHiとRmに同じレジスタを指定した場合、結果は予測できません。
RdLoとRmに同じレジスタを指定した場合、結果は予測できません。
RdhiとRdLoに同じレジスタを指定した場合、結果は予測できません。

アーリ・ターミネーション：乗算器インプリメンテーションが、アーリ・ターミネションをサポートしている場合、Rsオペランドの値にインプリメントしなければなりません。使用するアーリ・ターミネーションの種類（符号付きまたは符号なし）は、インプリメンテーションで定義されます。
