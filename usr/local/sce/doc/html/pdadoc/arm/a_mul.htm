ARM Architecture Reference Manual

MUL

機能説明

 乗算
アーキテクチャ アーキテクチャ v1ではリポートがない

MUL{<cond>}{<S>} Rd, Rm, Rs

説明

MUL(Multiply)命令は、符号付きまたは符号なし変数を乗算して、32ビットの結果を生成します。

MULは、レジスタRmの値とレジスタRsの値を乗算して、結果をデスティネーション・レジスタRdにストアします。コンディション・コード・フラグはオプションにより、（結果に基づいて）更新されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = (Rm * Rs)[31:0]
        if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = unpredictable
                V Flag = unaffected

修飾子

コンディション・コード

注記

R15の使用：レジスタRd、Rm、RsにR15を指定した場合、結果は予測できません。

オペランドの制限：RdとRmに同じレジスタを指定した場合、結果は予測できません。

アーリ・ターミネーション：乗算器インプリメンテーションが、アーリ・ターミネションをサポートしている場合、Rsオペランドの値にインプリメントしなければなりません。使用するアーリ・ターミネーションの種類（符号付きまたは符号なし）は、インプリメンテーションで定義されます。

符号付きおよび符号なし：MUL命令は64ビット積の下位32ビットしか生成しないため、MUL命令では符号付き数値と符号なし数値の乗算は同じ結果になります。
