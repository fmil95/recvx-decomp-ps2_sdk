ARM Architecture Reference Manual

Addressing Mode 5

機能説明

 Immediate post-indexed

[Rn], #+/-(<8_bit_offset>*4)

説明

このアドレッシング・モードは一連の連続アドレスを生成します。

最初のアドレスはベース・レジスタRnの値です。以降の連続したアドレスは、コプロセッサが命令終了を通知するまで、前のアドレスを4ずつインクリメントして生成されます。これによって、コプロセッサはコプロセッサ定義サイズのデータにアクセスすることができます。

ベース・レジスタRnは、ベース・レジスタRnの値にイミディエイト・オフセットの値×4を加算するか、またはベース・レジスタRnの値からイミディエイト・オフセットの値×4を減算することにより更新されます。

コプロセッサに16ワードを超えるデータ転送を要求してはなりません。

オペレーション

if ConditionPassed(<cond>) then
        <start_address> = Rn
        if U == 1 then
                Rn = Rn + <8_bit_offset> * 4
        else /* U == 0 */
                Rn = Rn - <8_bit_offset> * 4
        <address> = <start_address>
        while (NotFinished(coprocessor[<cp_num>]))
             <address> = <address> + 4
        <end_address> = <address>

注記

Nビット：このビットはコプロセッサに依存します。

Lビット：ロード命令(L==1)とストア命令(L==0)を区別します。

R15の使用：レジスタRnとしてR15を指定した場合、使用される値は命令のアドレスに8を加算した値です。

Wビット：ビット21(Writebackビット)がセットされていない場合、結果は予測できません。
