ARM Architecture Reference Manual

RSC

機能説明

 データ処理
アドレッシングモード アドレッシングモード１

RSC{<cond>}{S} Rd, Rn, <shifter_operand>

説明

RSC (Reverse Subtract with Carry) 命令は、<shifter_operand>の値からレジスタRnの値とNOT (Carry Flag)の値を減算し、結果をデスティネーション・レジスタRdにストアします。コンディション・コード・フラグは、オプションにより（結果に基づいて）更新されます。

R0、R1の64ビット値をネゲートするには、以下の順序に従って（R0は最下位ワードを保持する）、結果をR2、R3にストアします。

RSBS  R2, R0, #0
RSC   R3, R1, #0

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

31 - - 28 27 26 25 24 23 22 21 20 19 - - 16 15 - - 12 11 - - - - - - - - - - 0

オペレーション

if ConditionPassed(<cond>) then
        Rd = <shifter_operand> - Rn - NOT(C Flag)
        if S == 1 and Rd == R15 then
               CPSR = SPSR
        else if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = NOT BorrowFrom(<shifter_operand> - Rn - NOT(C Flag))
                V Flag = OverflowFrom(<shifter_operand> - Rn - NOT(C Flag))

例外

なし

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、Z、CおよびVを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16の「アドレッシング・モード1」で説明しています。

R15への書き込み：RdがR15で命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15でSフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

Iビット：ビット25を使用して、<shifter_operand> のイミディエイト形式とレジスタ形式を区別することができます。
