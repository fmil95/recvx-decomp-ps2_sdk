ARM Architecture Reference Manual

Addressing Mode 4

機能説明

 Decrement before

LDM|STM{<cond>}DB Rd{!}, <registers>{^}

説明

このアドレッシング・モードは、ロードおよびストア・マルチプル命令用で、アドレスの範囲を形成します。

最初に形成されるアドレスは<start_address>で、これはベース・レジスタRnの値から<register_list>に指定されたレジスタ数×4を減算した値です。以降のアドレスは前のアドレスを4だけインクリメントして形成されます。<register_list>に指定された各レジスタに対して1つのアドレスが生成されます。

最後に生成されるアドレスは<end_address>です。この値はベース・レジスタRnから4を減算した値です。

命令で指定された条件がコンディション・コードのステータスと一致し、Wビットがセットされている場合、Rnは<register_list>のレジスタ数×4だけデクリメントされます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

<start_address> = Rn - (Number_Of_Set_Bits_In(<register_list>) * 4)
<end_address> = Rn - 4
if ConditionPassed(<cond>) and W == 1 then
      Rn = Rn - (Number_Of_Set_Bits_In(<register_list>) * 4)

修飾子

！はWビットをセットし、ベース・レジスタを更新します。
＾はSビットをセットするのに使用します。下記参照。

注記

Lビット：ロード・マルチプルとストア・マルチプルを区別します。

Sビット：Sビットの作用は、3.19「ロードおよびストア・マルチプル・アドレッシング・モード｣で説明されています。
