ARM Architecture Reference Manual

B（1）

機能説明

 Conditional branch

アーキテクチャ

 アーキテクチャ v4T のみ

B<cond> <target_address>

説明

この形式のB（分岐）命令は、プログラム・フローを条件付きで変化させます。

この場合、Bによってターゲット・アドレスへの条件分岐が発生します。分岐先アドレスは、8ビットの符号付きオフセットを左に1ビットシフトし、結果を32ビットに符号拡張して、それをPCの内容（分岐命令のアドレス＋4が収められている）に加算することによって計算します。したがって、この命令は+/-256バイトの分岐を指定することができます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

コンディション・コード

注記

オフセット計算：アセンブラは、現在の命令のアドレスと分岐先アドレス（プログラム・ラベルとして与えられたもの）の差から4を減算して（PCは現在の命令のアドレス＋4を保持しているため）、分岐オフセットのアドレスを計算します。

メモリ境界：ロケーション・ゼロを超えて後方に分岐したり、32ビット・アドレス空間の上限を超えて分岐した場合、結果は予測できません。
