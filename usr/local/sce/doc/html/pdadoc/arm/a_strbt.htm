ARM Architecture Reference Manual

STRBT

機能説明

 ロード／ストア
アドレッシングモード アドレッシングモード2

STR{<cond>}BT Rd, <post_indexed_addressing_mode>

説明

STRBT (Store Register Byte with Translation) 命令は、通常ユーザ・モードで実行されるメモリ・アクセス命令をエミュレートしている（特権付き）例外ハンドラが使用できます。アクセスはユーザ・モード特権のように制限されます。

STRBTは、レジスタRdの最下位バイトから<post_indexed_addressing_mode>で計算されるメモリのアドレスに1バイトをストアします。プロセッサが特権モードにあるときにこの命令が実行されると、メモリ・システムにこのアクセスをプロセッサがユーザ・モードにあるときと同様に処理するよう通知されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、およびUビットは、<addressing_mode>のタイプを指定します(セクション3.17の「アドレッシング・モード2」を参照)。

レジスタRn：<post_indexed_addressing_mode>で使用するベース・レジスタを指定します。

ユーザ・モード：この命令がユーザ・モードで実行される場合、通常のユーザ・モードのアクセスが実行されます。

R15の使用：Rdにレジスタ15を指定した場合、結果は予測できません。

オペランド制限：RdおよびRnを同じレジスタに指定した場合、結果は予測できません。

データ・アボート：データ・アボートが通知された場合、Rnに残る値はインプリメンテーションで定義されますが、元のベース・レジスタの値または更新されたベース・レジスタの値のいずれかになります（RdとRnに同じレジスタを指定した場合でも）。
