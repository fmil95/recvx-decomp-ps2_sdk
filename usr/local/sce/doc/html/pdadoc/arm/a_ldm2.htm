ARM Architecture Reference Manual

LDM（2）

機能説明

 ロード／ストアマルチプル

アドレッシングモード

 アドレッシングモード4

LDM{<cond>}<addressing_mode> Rn, <registers>^

説明

この形式のLDM（Load Multiple）命令は、プロセッサが（プロセスのスワップを実行する場合便利なように）特権モードになっている場合に、ユーザ・モードのレジスタをロードします。

この場合、LDM命令はシーケンシャル・メモリ・ロケーションからユーザ・モードの汎用レジスタ（システム・モードの汎用レジスタでもある）の空でないサブセット（またはPCを除くすべてのセット）をロードします。これらのレジスタは、最小番号のレジスタを先頭に最下位メモリ・アドレス（<start_address>）から、または最大番号のレジスタを最後に最上位メモリ・アドレス（<end_address>）から、という順序でロードされます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：P、Uビットは、異なるタイプのアドレッシング・モードを区別します。セクション3.19の「アドレッシング・モード4」を参照してください。

バンクされたレジスタ：LDMの後にバンクされたレジスタにアクセスする命令が続いてはなりません（これを保証するには、後にNOPを続けるのがよい方法です）。

ライトバック：ビット21（Wビット）をセットした場合、結果は予測できません。

ユーザおよびシステム・モード：LDMはユーザ・モードおよびシステム・モードでは予測不能です。

レジスタRn：<addressing_mode>で使用するベース・レジスタを指定します。

R15の使用：レジスタ15をベース・レジスタRnとして指定する場合、結果は予測できません。

ベース・レジスタ・モード：ベース・レジスタは、ユーザ・モードのレジスタではなく、現在のプロセッサ・モードのレジスタから読み出されます。

データ・アボート：データ・アボートが通知された場合、Rnに残った値は元のベース・レジスタ値になります。

ワードに整列していないアドレス：LDM命令は、<address>の最下位2ビットを無視します（ワードはLoad Wordのようにローテートされません）。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
