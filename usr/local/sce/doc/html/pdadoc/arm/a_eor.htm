ARM Architecture Reference Manual

EOR

機能説明

 データ処理

アドレッシングモード

 アドレッシングモード１

アーキテクチャ

 アーキテクチャ v4T のみ

EOR{<cond>}{S} Rd, Rn, <shifter_operand>

説明

EOR（Exclusive-OR）命令は、レジスタRnの値と<shifter_operand>の値で、ビット単位の排他的論理ORを実行して、結果をデスティネーション・レジスタRdにストアします。オプションにより、（結果に基づいて）コンディション・コード・フラグが更新されます。

EORは、レジスタの選択されたビットを反転するのに使用できます。各ビットごとに、1と指定されたビットは反転し、0と指定されたビットは変化しません。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = Rn EOR <shifter_operand>
        if S == 1 and Rd == R15 then
                CPSR = SPSR
        else if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = <shifter_carry_out>
                V Flag = unaffected

例外

なし

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、Z、Cを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16以降の「アドレッシング・モード1」で説明しています。

R15への書き込み：RdがR15で、命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15で、Sフラグがセットされている場合、オペレーションの結果がPCに置かれ、現在のモードに対応するSPSRはCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変更を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

Iビット：ビット25を使用して、<shifter_operand>のイミディエイト形式とレジスタ形式を区別することができます。
