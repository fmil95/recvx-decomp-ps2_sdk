ARM Architecture Reference Manual

RSB

機能説明

 データ処理
アドレッシングモード アドレッシングモード１

RSB{<cond>}{S} Rd, Rn, <shifter_operand>

説明

RSB (Reverse Subtract)命令は、<shifter_operand>の値からレジスタRnの値を減算し、結果をデスティネーション・レジスタRdにストアします。コンディション・コード・フラグは、オプションにより（結果に基づいて）更新されます。

以下の命令はRxの負数の値（2の補数）をRdにストアします。

RSB Rd、Rd、Rx、#0

（Rxの値に）2n-1を定数乗算する（結果をRdに格納）場合は、以下で実行できます。

RSB Rd, Rx, Rx LSL #n

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = <shifter_operand> - Rn
        if S == 1 and Rd == R15 then
                CPSR = SPSR
        else if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = NOT BorrowFrom(<shifter_operand> - Rn)
                V Flag = OverflowFrom(<shifter_operand> - Rn)

例外

なし

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、Z、CおよびVを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、「アドレッシング・モード1」で説明しています。

R15への書き込み：RdがR15で命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15でSフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

Iビット：ビット25を使用して、<shifter_operand> のイミディエイト形式とレジスタ形式を区別することができます。
