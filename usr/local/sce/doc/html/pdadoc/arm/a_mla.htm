ARM Architecture Reference Manual

MLA

機能説明

 乗算

アーキテクチャ

 アーキテクチャ v１ではサポートされない

MLA{<cond>}{<S>} Rd, Rm, Rs, Rn

説明

MLA (Multiply Accumulate)命令は、符号付きオペランド、または符号なしオペランドを乗算して、32ビットの結果を生成し、第3オペランドに加算した後、デスティネーション・レジスタに書き込みます。

MLAは、レジスタRmの値とレジスタRsの値を乗算して、レジスタRnの値に加算し、その結果をデスティネーション・レジスタRdにストアします。オプションにより、（結果に基づいて）コンディション・コード・フラグが更新されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = (Rm * Rs + Rn)[31:0]
        if S == 1 then
                N Flag = Rd[31]
                Z Flag = if Rd == 0 then 1 else 0
                C Flag = unpredictable
                V Flag = unaffected

修飾子

コンディション・コード
Sはコンディション・コード・フラグNとZを更新します。

注記

R15の使用：レジスタRd、Rm、Rs、またはRnにR15を指定した場合、結果は予測できません。

オペランドの制限：RdとRmに同じレジスタを指定した場合、結果は予測できません。

アーリ・ターミネーション：乗算器インプリメンテーションが、アーリ・ターミネションをサポートしている場合、Rsオペランドの値にインプリメントしなければなりません。使用するアーリ・ターミネーションの種類（符号付きまたは符号なし）は、インプリメンテーションで定義されます。

符号付き、符号なし：MLA命令は64ビット積の下位32ビットしか生成しないため、符号付きの値、符号なしの値のいずれの乗算も同じ結果になります。
