ARM Architecture Reference Manual

SWP

機能説明

 セマフォ
アーキテクチャ アーキテクチャｖ１、ｖ２ではリポートされない

SWP{<cond>} Rd, Rm, [Rn]

説明

SWP(Swap) 命令は、レジスタとメモリ間でワードをスワップします。

SWPは、レジスタRnの値で与えられるメモリ・アドレスから1ワードをロードします。レジスタRmの値は、Rnの値で与えられるメモリ・アドレスにストアし、そのアドレスにロードされていた値をレジスタRdに書き込みます。RdとRnに同じレジスタを指定した場合、この命令はレジスタの値とメモリ・アドレスの値をスワップします。
この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

コンディション・コード

注記

ワードに整列していないアドレス：アドレスがワードに整列していない場合、ロードされた値は、<address> [1:0]の値×8だけ右にローテートされます。

R15の使用：Rd、Rn、またはRmにレジスタ15を指定した場合、結果は予測できません。

オペランドの制限：RnとRm、またはRnとRdに同じレジスタを指定した場合、結果は予測できません。

データ・アボート：ロード・アクセスまたはストア・アクセスのいずれか（または両方）でデータ・アボートが通知された場合、ロードされた値はRdに書き込まれません。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
