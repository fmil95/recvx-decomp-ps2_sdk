ARM Architecture Reference Manual

STM

機能説明

 ロード／ストア・マルチプル
アドレッシングモード アドレッシングモード４

STM{<cond>}<addressing_mode> Rn{!}, <registers>

説明

STM (Store Multiple) 命令は、ブロック・ストア命令 (効率的なブロック・コピーが可能なロード・マルチプル命令と組み合わせて）として、また汎用レジスタとリターン・アドレスをセーブするためのプロシージャ・エントリなどのスタック操作やスタック・ポインタの更新に役立ちます。

STMは、汎用レジスタの空でないサブセット（または全セット）を順次メモリ・ロケーションにストアします。最小番号レジスタを先頭に最下位メモリ・アドレス（<start_address>）から順次ストアされ、最後に最大番号レジスタが最上位アドレス(<end_address>)にストアされます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレシング・モード：P、U、およびWビットは、異なるタイプのアドレッシング・モードを区別します。セクション3.19の「アドレッシング・モード4」を参照してください。

レジスタRn：<addresing_mode>で使用しているべース・レジスタを指定します。

R15の使用：レジスタ15をベース・レジスタとして指定した場合、結果は予測できません。レジスタ15を<register_list>に指定し、ライトバックを指定した場合、ストアされる値はインプリメンテーションで定義されたものです。

オペランドの制限：Rnを<register_list>に指定し、ライトバックを指定した場合、Rnにストアされる値は予測できません。

データ・アボート：データ・アボートが通知され、<addressing_mode>でライトバックを指定した場合、Rnに残る値はインプリメンテーションで定義されますが、元のベース・レジスタの値または更新されたベース・レジスタの値のいずれかになります。

ワードに整列していないアドレス：STM命令は、<address>の最下位2ビットを無視します（ワードはLoad Word命令のようにローテートされません）。

アラインメント：インプリメンテーションにシステム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
