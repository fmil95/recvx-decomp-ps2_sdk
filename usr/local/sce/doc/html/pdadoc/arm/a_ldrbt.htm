ARM Architecture Reference Manual

LDRBT

機能説明

 ロード／ストア

アドレッシングモード

 アドレッシングモード2

LDR{<cond>}BT Rd, <post_indexed_addressing_mode>

説明

LDRBT（Load Register Byte with Transition）命令は、通常ユーザ・モードで実行するメモリ・アクセス命令をエミュレートする（特権）例外ハンドラで使用することができます。このアクセスは、ユーザ・モード特権を持っている場合と同じように制限されます。

LDRBTは、<post_indexed_addressing_mode>によって計算されたメモリ・アドレスから1バイトをロードし、これを32ビット・ワードにゼロ拡張して、レジスタRdに書き込みます。プロセッサが特権モードの場合にこの命令を実行すると、メモリ・システムに対し、プロセッサがユーザ・モードのときと同じようにこのアクセスを処理するよう通知されます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、Uビットは、<addressing_mode>のタイプを指定します（セクション3.17の「アドレッシング・モード2」を参照してください）。

レジスタRn：<post_indexed_addressing_mode>で使用したベース・レジスタを指定します。

ユーザ・モード：この命令がユーザ・モードで実行された場合、通常のユーザ・モードのアクセスが行われます。

R15の使用：Rdにレジスタ15が指定された場合、結果は予測できません。

オペランドの制限：RdとRnに同じレジスタが指定された場合、結果は予測できません。

データ・アボート：データ・アボートが通知された場合、Rnに残った値はインプリメンテーションで定義されますが、（RdとRnに同じレジスタが指定されている場合でも）ものとベース・レジスタ値か更新されたベース・レジスタ値になります。
