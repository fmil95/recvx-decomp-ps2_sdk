ARM Architecture Reference Manual

LDRH（1）

機能説明

 Load unsigned halfword, immediate offset

アーキテクチャ

 アーキテクチャ v4T のみ

LDRH Rd, [Rn, #<5_bit_offset> * 2]

説明

この形式のLDRH（Load Register Halfword）命令は、16ビットのメモリ・データを汎用レジスタにロードし、その値を操作することができます。このアドレッシング・モードは、構造体（レコード）にアクセスするのに便利です。オフセットがゼロの場合、生成されるアドレスはベース・レジスタのRnと同じです。

この場合、LDRHは以下の操作を行います。

  1. メモリから1ハーフワードをロードし、
  2. このハーフワードを32ビット・ワードにゼロ拡張して、
  3. これをレジスタRdに書き込みます。

メモリ・アドレスは、<5_bit_offset>の値×2をレジスタRnの値に加算して計算されます。アドレスがハーフワードに整列していない場合、結果は予測できません。

注記

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章「システム・アーキテクチャとシステム・コントロール・コプロセッサ」を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[0]!=0のアドレスによって、アラインメント例外が発生します。
