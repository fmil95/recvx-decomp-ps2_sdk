ARM Architecture Reference Manual

LDR

機能説明

 ロード／ストア

アドレッシングモード

 アドレッシングモード2

LDR{<cond>} Rd, <addressing_mode>

説明

LDR（Load register）命令は、適切なアドレッシング・モードと組み合わせると、32ビットのメモリ・データを汎用レジスタにロードし、汎用レジスタでその値を操作することができます。デスティネーション・レジスタがPCの場合、この命令はメモリから32ビット・アドレスをロードし、そのアドレスに分岐します（LDR命令の前に、MOV LR, PCを置き、分岐とリンクを合成します）。

PCをベース・レジスタとして使用すると、PC相対アドレッシングが可能で、容易に位置独立コードを生成することができます。

LDRは<addressing_mode>で計算されたメモリ・アドレスから1ワードをロードし、それをレジスタRdに書き込みます。アドレスがワードに整列していない場合、ロードされたデータは、アドレス指定されたバイトがこのレジスタの最下位バイトに入るようにローテートされます。PCをレジスタRdとして指定した場合、この命令はアドレス（データ）への分岐をPCにロードします。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

注記

アドレッシング・モード：I、P、U、Wビットは、<addressing_mode>のタイプを指定します（セクション3.17「アドレッシング・モード2」を参照してください）。

レジスタRn：<addressing_mode>で使用したベース・レジスタを指定します。

データ・アボート：データ・アボートが通知され、<addressing_mode>がプレインデックスまたはポストインデックス付きアドレッシングを使用する場合、Rnに残った値は、インプリメンテーションで定義されますが、（RdとRnに同じレジスタが指定されている場合でも）元のベース・レジスタ値か更新されたベース・レジスタ値になります。

オペランドの制限：<addressing_mode>がプレインデックスまたはポストインデックス付きアドレッシングを使用し、RdとRnに同じレジスタが指定されている場合、結果は予測できません。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
