ARM Architecture Reference Manual

Addressing Mode 5

機能説明

 Immediate offset

[Rn, #+/-(<8_bit_offset>*4)]

説明

このアドレッシング・モードは一連の連続アドレスを生成します。

最初のアドレスはベース・レジスタRnの値にイミディエイト・オフセットの値×4を加算するか、ベース・レジスタRnの値から、イミディエイト・オフセットの値×4を減算して計算されます。以降の連続したアドレスは、コプロセッサが命令終了を通知するまで、前のアドレスを4ずつインクリメントして生成されます。これによって、コプロセッサはコプロセッサ定義サイズのデータにアクセスすることができます。

コプロセッサに16ワードを超えるデータ転送を要求してはなりません。

オペレーション

if ConditionPassed(<cond>) then
        if U == 1 then
                <address> = Rn + <8_bit_offset> * 4
        else /* U == 0 */
                <address> = Rn - <8_bit_offset> * 4
        <start_address> = <address>
        while (NotFinished(coprocessor[<cp_num>]))
                <address> = <address> + 4

注記

Nビット：このビットはコプロセッサに依存します。

Lビット：ロード命令(L==1)とストア命令(L==0)を区別します。

R15の使用：レジスタRnとしてR15を指定した場合、使用される値は命令のアドレスに8を加算した値です。
