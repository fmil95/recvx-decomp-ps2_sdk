ARM Architecture Reference Manual

MRC

機能説明

 ロード／ストアマルチプル

アドレッシングモード

 アドレッシングモード4

MRC{<cond>} p<cp_num>, <opcode_1>, Rd, CRn, CRm, <opcode_2>

説明

MRC (Move to ARM Register from Coprocessor) 命令は、ARMレジスタに値を返すコプロセッサ命令を開始するために使用されます。たとえば、浮動小数点アクセラレータ・コプロセッサ用の浮動少数点から固定少数点の変換命令です。

R15をデスティネーション・レジスタとして指定すると、浮動小数点比較命令などのオペレーションに便利です。

MCRには以下の2つの用途があります：

  1. Rdがレジスタ15を指定する場合、コンディション・コードのフラグ・ビットは、<cp_num>で指定されるコプロセッサの上位4ビットの値から更新され（コプロセッサのステータスに基づく条件分岐を可能にするため)、他の28ビットは無視されます。
  2. それ以外の場合、命令はレジスタRdに<cp#>で指定されるコプロセッサからの値を書き込みます。

命令を実行できることを示すコプロセッサがない場合、未定義命令例外が発生します。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        if Rd == 15 then
                N flag = (value from Coprocessor[<cp_num>])[31]
                Z flag = (value from Coprocessor[<cp_num>])[30]
                C flag = (value from Coprocessor[<cp_num>])[29]
                V flag = (value from Coprocessor[<cp_num>])[28]
        else /* Rd != 15 */
                Rd = value from Coprocessor[<cp_num>]
