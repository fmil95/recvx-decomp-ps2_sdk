ARM Architecture Reference Manual

PUSH

機能説明

 Push multiple registers

アーキテクチャ

 アーキテクチャ v4T のみ

PUSH {<register_list>, <LR>}

説明

PUSH（Push Multiple Registers）命令は、プロシージャ・エントリを含むスタック操作で、レジスタをセーブし（オプションでリターン・アドレスを含む）、スタック・ポインタを更新するのに便利です。

PUSHは、汎用レジスタのサブセット（または完全なセット）をストアし、オプションで順次メモリ・ロケーションにLRをストアします。レジスタは以下の順序でストアされます。

  • 最小番号のレジスタを先頭に、最下位メモリ・アドレス（<start_address>）から
  • 最大番号のレジスタを最後に、最上位メモリ・アドレス（<end_address>）から

<start_address>はSPの値です。

これ以降のアドレスは、前のアドレスを4ずつインクリメントして形成されます。<register_list>で指定された各レジスタに対して、1つのアドレスが生成されます。

<end_address>の値は、SPの値と<register_list>で指定したレジスタ数（Rビットを含む）×4の合計から4を引いたものです。
最後に、ベース・レジスタRnを、<register_list>のレジスタ数（＋Rビット）×4だけインクリメントします。

オペレーション

<start_address> = SP
<end_address> = SP+(Number_Of_Set_Bits_In(<register_list>+R)*4)-4
<address> = <start_address>
for i = 0 to 7
	if <register_list>[i] == 1
		Memory[<address>,4] = Ri
		<address> = <address> + 4
	if R == 1
		Memory[<address>,4] = LR
		<address> = <address> + 4
assert <end_address> == <address> - 4
SP = SP - 4*(R + Number_Of_Set_Bits_In(<register_list>))

例外

データ・アボート

修飾子

なし

注記

Rビット：R==1の場合、LRもスタックにストアされます。R==0の場合、LRはストアされません。

データ・アボート：データ・アボートが通知された場合、SPに残った値はインプリメンテーションで定義されますが、もとのSP値か更新されたSP値になります。

ワードに整列していないアドレス：Push multiple命令は、<address>の最下位2ビットを無視します（ワードはLoad Wordのようにローテートされません）。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章「システム・アーキテクチャとシステム・コントロール・コプロセッサ」を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。

同等なARM構文とエンコーディング

STMDB SP!, {<register_list>, <LR>}
