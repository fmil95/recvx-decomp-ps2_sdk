ARM Architecture Reference Manual

LDM（3）

機能説明

 ロード／ストアマルチプル

アドレッシングモード

 アドレッシングモード4

LDM{<cond>}<addressing_mode> Rn{!}, <registers_and_pc>^

説明

この形式のLDM（Load Multiple）命令は、例外からのリターンで、セーブされたレジスタのリストア、PCへのリターン・アドレスのロード、スタック・ポインタの更新、およびSPSRからのCPSRのリストアに便利です。

この場合、LDMはシーケンシャル・メモリ・ロケーションから汎用レジスタの空でないサブセット（または完全なセット）とPCをロードします。これらのレジスタは、最小番号のレジスタを先頭に最下位メモリ・アドレスから、または最大番号のレジスタを最後に最上位メモリ・アドレスから、という順序でロードされます。現在のモードのSPSRは、CPSRにコピーされます。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致する場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

if ConditionPassed(<cond>) then
        <address> = <start_address>

コンディション・コード
!はWビットをセットし、ベース・レジスタを更新します。

注記

アドレッシング・モード：P、U、Wビットは、異なるタイプのアドレッシング・モードを区別します。セクション3.19「アドレッシング・モード4」を参照してください。

レジスタRn：<addressing_mode>で使用したベース・レジスタを指定します。

R15の使用：R15をベース・レジスタRnとして使用した場合、結果は予測できません。

ユーザおよびシステム・モード：この命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

オペランドの制限：ベース・レジスタRnが<register_list>に指定され、ライトバックが指定されている場合、Rnの最終値は予測できません。

データ・アボート：データ・アボートが通知され、<addressing_mode>がライトバックを指定している場合、Rnに残った値はインプリメンテーションで定義されていますが、（Rnが<register_list>に指定されている場合でも）元のベース・レジスタ値か更新されたベース・レジスタ値です。レジスタ15が<register_list>に指定されている場合、データ・アボートが発生した場合にこれを上書きしてはなりません。

ワードに整列していないアドレス：Load multiple命令は、<address>の最下位2ビットを無視します（ワードはLoad Wordのようにローテートされません）。

アラインメント：インプリメンテーションに、システム・コントロール・コプロセッサ（英文リファレンスマニュアル第7章を参照）が含まれており、アラインメント・チェックがイネーブルされている場合、ビット[1:0]!=0b00のアドレスによって、アラインメント例外が発生します。
