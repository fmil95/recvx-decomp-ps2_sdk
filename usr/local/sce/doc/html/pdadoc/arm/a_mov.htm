MOV

機能説明

 データ処理

アドレッシングモード

 アドレッシングモード１

MOV{<cond>}{S} Rd, <shifter_operand>

説明

MOV (Move) 命令には、以下の用途があります。

  • あるレジスタから別のレジスタに値を移動する。
  • レジスタに定数値を入力する。
  • 他の算術または論理オペレーションなしで、シフトを実行する。

PCが命令のデスティネーションの場合は分岐が発生し、MOV PC, LRを使用して、サブルーチン・コールからリターンしたり（B命令およびBL命令を参照）、何種類かの例外からリターンすることができます（2.5「例外」を参照）。

MOVは、<shifter_operand>の値をデスティネーション・レジスタRdに移動し、オプションにより（結果に基づいて）コンディション・コード・フラグを更新します。

この命令は、命令で指定された条件がコンディション・コードのステータスと一致した場合にのみ実行されます。条件は、3.3「コンディション・フィールド」で定義されています。

オペレーション

if ConditionPassed(<cond>) then
        Rd = <shifter_operand>
        if S == 1 and Rd == R15 then
                CPSR = SPSR
        else if S == 1 then

                Z Flag = if Rd == 0 then 1 else 0
                C Flag = <shifter_carry_out>
                V Flag = unaffected

例外

修飾子

コンディション・コード
Sはコンディション・コード・フラグN、ZおよびCを更新します。

注記

シフタ・オペランド：この命令のシフタ・オペランドは、セクション3.16の「アドレッシング・モード1」で説明しています。

R15への書き込み：RdがR15で命令のSフラグがセットされていない場合、オペレーションの結果はPCに置かれます。RdがR15でSフラグがセットされている場合、オペレーションの結果はPCに置かれ、現在のモードに対応するSPSRがCPSRに転送されます。これによって、PCとCPSRの両方をアトミックにリストアする状態変化を実現できます。この形式の命令は、ユーザ・モードおよびシステム・モードでは予測不能です。

Iビット：ビット25を使用して、<shifter_operand> のイミディエイト形式とレジスタ形式を区別することができます。
