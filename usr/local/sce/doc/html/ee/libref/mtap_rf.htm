関数

sceMtapChangeThreadPriority

IOPのスレッドプライオリティを変更する

形式 

int sceMtapChangeThreadPriority( 
	int priority_high,
	int priority_low
);
引数 
priority_high	メインスレッドのプライオリティ
priority_low	SIFインターフェイススレッドのプライオリティ
返り値
変更成功
1以外 失敗

解説
IOPのモジュールmtapman.irxのスレッドプライオリティを変更します。スレッドプライオリティは、
priority_high メインスレッド
priority_low SIFインターフェイススレッド
の両方を指定してください。メインスレッドは１フレームに１回実行されます。SIFインターフェイススレッドは、
sceMtapPortOpen()
sceMtapPortClose()
sceMtapGetConnection()
sceMtapChangeThreadPrioriry()
実行時に起動します。
なお、スレッドプライオリティはmtapman.irxをロードする時に指定することもできます。
この関数は、sceMtapInit()実行後でなければ実行できません。

関連項目

sceMtapGetConnection

マルチタップの接続状態の取得

形式 

int sceMtapGetConnection( 
	int port 
)
引数 
port	ポート番号
返り値
1：マルチタップあり
1以外： マルチタップなし

解説
オープンされたポートに対して、マルチタップが接続されているかどうかの情報を取得します。この関数はRPCを使用して最新の情報をIOPに問い合わせます。
オープンされていないポートに付いては、マルチタップの接続に関係なく"マルチタップなし"が返されます。

sceMtapPortClose

使用するマルチタップが接続されるポートをクローズする

形式 

int sceMtapPortClose( 
	int port 
)
引数 
port	ポート番号
	コントローラポート：0または1
	メモリーカードスロット：	2または3
返り値
1：リクエストは受理された
1以外：リクエストは受理されなかった

解説
マルチタップの接続先としてオープンしたポートをクローズします。
なお、コントローラポート0とメモリーカードスロット2(またはコントローラポート1とメモリーカードスロット3)は物理的には同じマルチタップが接続されていますが、内部では独立したマルチタップとして動作しています。
よって、sceMtapPortOpen()でオープンしたポートはそれぞれクローズしてください。
関連項目
sceMtapPortOpen()

sceMtapPortOpen

使用するマルチタップが接続されるポートをオープンする

形式 

int sceMtapPortOpen( 
	int port 
)
引数 
port	ポート番号
	コントローラポート：0または1
	メモリーカードスロット：	2または3
返り値
1：リクエストは受理された
1以外：リクエストは受理されなかった

解説
マルチタップの接続先として監視を行うポートを指定します。
IOPのドライバは、この関数で指定されたポートのみをマルチタップの接続先として認識し、定期的に状態をポーリングします。
なお、コントローラポート0とメモリーカードスロット2(またはコントローラポート1とメモリーカードスロット3)は物理的には同じマルチタップが接続されていますが、内部では独立したマルチタップとして動作しています。
よって、コントローラ、メモリーカード両方のポートのマルチタップを使用する場合は、両方オープンしてください。オープンしない方は、差し込み口Aのみ有効になります。
