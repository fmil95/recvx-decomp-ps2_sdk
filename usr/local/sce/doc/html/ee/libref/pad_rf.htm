引数

scePadEnterPressMode

コントローラを感圧モードにする

形式 

int scePadEnterPressMode( 
	int port, 
	int slot 
)
引数 
port	コントローラポート番号
slot	スロット番号（マルチタップ使用時以外は0固定）
返り値
1：リクエストは受理された

scePadGetState

コントローラの接続状態取得

形式 

int scePadGetState(
	int port, 
	int slot 
)
引数 
port	コントローラポート番号
slot	スロット番号（マルチタップ使用時以外は０固定）
返り値
コントローラの接続状態

返り値 内容
scePadStateDiscon コントローラは接続されていない
scePadStateFindPad コントローラが見つかった（処理は継続中）
scePadStateFindCTP1 CTP1.0のコントローラを検出
scePadStateExecCmd コントローラと通信中
scePadStateStable CTP2.0のコントローラを検出
scePadStateError 通信エラー中 



    上記の状態のうち、ボタンの状態を得たり、アプリケーションからの 

    リクエストが可能なのはscePadStateFindCTP1およびscePadStateStableです。
解説
オープンしたポートの、コントローラの接続状態を取得します。
ポートをオープンすると、IOPのpadman.irxは独自にコントローラの接続を監視し、接続されたコントローラの情報を取得します。また、コントローラの状態を変更するには数回通信を行う必要があり、終了までに数フレームの時間を要します。
この間はボタンの状態を得ることができず、またアプリケーションからのリクエストを受け付けることができません。アプリケーションはscePadGetState()を使用してpadman.irxの処理の状態を調べることができます。scePadGetState()がscePadStateStable か scePadStateFindCTP1 であれば、コントローラからのボタン情報の取得ができます。

この関数は、scePadGetState()がscePadStateStableの時のみ有効となります。

scePadSetActAlign

コントローラに送信するアクチュエータパラメータの内訳設定

形式 

int scePadSetActAlign( 
	int port, 
	int slot, 
	unsigned char* data 
)
引数 
port	コントローラポート番号
slot	スロット番号（マルチタップ使用時以外は０固定）
data	アクチュエータパラメータの送信内訳（６バイト）
返り値
1：リクエストが受理された
1以外：リクエストは受理されない

解説
送信バッファの中でアクチュエータパラメータを設定する位置を、６バイト配列の中にアクチュエータ番号を書き込むことでコントローラに通知します。
送信バッファのオフセット０をアクチュエータ番号０に、オフセット１をアクチュエータ番号１に使い、そのほかのデータは使わないときの設定例は、以下のようになります（送信したい場所にアクチュエータ番号を、使わない場所には0xffを設定します）。

オフセット 0 1 2 3 4 5
dataの内容 0x00 0x01 0xFF 0xFF 0xFF 0xFF


この関数を実行するとコントローラと通信を行うため、数フレームの間、他のリクエストは受け付けられません。
処理の終了はscePadGetState()で監視しするか、scePadGetReqState()の結果で判定してください。
この関数で設定したアクチュエータパラメータ内訳は、コントローラが抜かれるか、コントローラモードが変更されると無効になります。

scePadSetActDirect

コントローラにアクチュエータパラメータを送信する

形式 

int scePadSetActDirect (
	int port, 
	int slot, 
	unsigned char* data
)
引数 
port	コントローラポート番号
slot	スロット番号（マルチタップ使用時以外は０固定）
data	送信データの先頭アドレス
	送信データは６バイト用意してください
返り値
１：転送成功
１以外：失敗

解説
アクチュエータを動作させるための送信データをIOPに転送します。転送された送信データは、次のVBlank割り込みでコントローラに送信されます。
アクチュエータを制御する場合はscePadSetActDirect()で送信バッファを指定するだけでなく、scePadSetActAlign()を用いて、送信バッファのどのオフセットにあるデータをアクチュエータパラメータとして認識すべきなのかをコントローラに対して通知しておく必要があります。
アクチュエータが受け付けられるデータ長はscePadInfoAct()によって調べることができ、DUALSHOCK2の場合アクチュエータ番号0,1のデータ長はそれぞれ1ビットと1バイトであることが分かります。従ってscePadSetActAlign()によって送信バッファのオフセット0にアクチュエータ番号0のパラメータを、オフセット1にアクチュエータ番号1のパラメータを送信する設定をしたとするとオフセット0には0または1、オフセット1には0?255の値を送ることによってアクチュエータを制御できることになります。
なおアクチュエータの動作はパラメータの値が0のとき停止状態で、値が大きくなるほど回転が速くなります。
DUALSHOCK2のアクチュエータ設定値
小モータ 0=停止, 1=回転
大モータ 0～255 数値が大きいほど回転数が速い


scePadSetMainMode

コントローラモード切り替え／切り替えスイッチのロック

形式 

int scePadSetMainMode( 
	int port, 
	int slot, 
	int offs, 
	int lock 
)
引数 
port	コントローラポート番号
slot	スロット番号（マルチタップ使用時以外は０固定）
offs	切り替えるコントローラモードが入っている
	コントローラモードIDテーブル内のオフセット
lock	アナログ切り替えスイッチのLock/Unlock指定
	0, 1：切り替えスイッチのLock/Unlockは現状維持
	2：Unlock
	3：Lock
返り値
1：リクエストは受理された
1以外：リクエストは受理されなかった

解説
コントローラモードの切り替え、およびコントローラ本体についているモード切り替えボタンのロック／アンロックを切り替えます。
この関数を実行すると、コントローラと通信を行うため数フレームの間は他のリクエストは受け付けられません。処理の終了はscePadGetState()で監視しするか、scePadGetReqState()の結果で判定してください。また、コントローラモードが切り替えられると、以前に設定していたアクチュエータ設定は無効となります。
